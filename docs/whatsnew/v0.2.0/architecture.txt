FILE: moewatch/config.py
────────────────────────────────────────────────────────────────────────────────

PURPOSE
  Central configuration dataclass. All tunable parameters in one place.
  This is the single source of truth for all configuration options.

ENUMS

  OutputMode (enum)
    - SILENT       : No output
    - CLI          : ANSI terminal output
    - JSON         : Newline-delimited JSON

  AlertLevel (enum)
    - INFO         : Informational
    - WARNING      : Warning-level alert
    - CRITICAL     : Critical alert

DATACLASS: WatchConfig
────────────────────────────────────────────────────────────────────────────────
  Fields (all with type hints and defaults):

  SIGNAL THRESHOLDS
    dead_threshold: float = 0.01
      Gradient norm below this → expert marked DEAD

    cold_threshold: float = 0.05
      Gradient norm below this → expert marked COLD

    cold_steps_limit: int = 50
      Consecutive steps as COLD before promotion to DEAD

    entropy_warn: float = 0.3
      Normalized entropy below this → WARNING level

    entropy_critical: float = 0.15
      Normalized entropy below this → CRITICAL level

    entropy_drop_warn: float = 0.1
      Entropy drop rate above this (per step) → WARNING

    load_imbalance_warn: float = 3.0
      Load imbalance ratio (max/mean) above this → WARNING

    load_imbalance_error: float = 5.0
      Load imbalance ratio above this → CRITICAL

  COLLECTION & MONITORING
    log_every: int = 10
      Emit reports every N steps

    sample_every: int = 10
      Sample gradient hooks every N steps (to reduce overhead)

    output: OutputMode = OutputMode.CLI
      Output format: SILENT, CLI, or JSON

  BEHAVIOR
    no_color: bool = False
      Disable ANSI colors in CLI output

    router_modules: list[str] | None = None
      Manual override for router module names if auto-detection fails

    intervention_enabled: bool = True
      Enable intervention engine

    policy_type: str = "rule"
      "rule" (Phase 1) or "bandit" (Phase 2)

  POLICY PARAMETERS (Phase 2 Bandit)
    bandit_epsilon: float = 0.1
      Exploration rate (epsilon-greedy)

    reward_discount_gamma: float = 0.95
      Discount factor for delayed rewards

    reward_window_steps: int = 50
      Observation window for reward computation

  BASELINE TRACKER
    baseline_min_clean_steps: int = 20
      Minimum clean (non-intervention) history steps before baseline valid

    baseline_exclusion_window: int = 100
      Steps to exclude after intervention before using baseline again

  INTERVENTION SAFETY
    intervention_cooldown: int = 200
      Minimum steps between interventions on same layer

    intervention_max_delta: float = 0.1
      Maximum change to any hyperparameter per intervention

    loss_guard_threshold: float = 1.5
      Spike factor above baseline loss to freeze interventions

METHODS

  __post_init__() → None
    Validates all config values on construction. Raises ValueError if invalid.

DEPENDENCIES
  None (pure dataclass, no external deps)

IMPLEMENTATION NOTES
  - Use @dataclass decorator from dataclasses module
  - All numeric fields should have sensible defaults
  - Implement __post_init__() to validate:
    * 0.0 < entropy_warn < 1.0
    * cold_threshold > dead_threshold
    * entropy_drop_warn > 0
    * policy_type in ["rule", "bandit"]
    * output in OutputMode enum values

FILE: moewatch/_watcher.py
────────────────────────────────────────────────────────────────────────────────

PURPOSE
  Main MoEWatch class. Manages full lifecycle of live training-time monitoring.
  Coordinates HookManager, StatCollector, all analyzers, intervention engine,
  and policy. This is the primary user-facing class.

CLASS: MoEWatch
────────────────────────────────────────────────────────────────────────────────

  DOCSTRING
    """
    Live training-time MoE routing collapse monitor and intervention controller.

    Attaches to a HuggingFace Trainer and monitors routing health at each
    training step. Emits alerts and can apply corrective interventions to live
    training hyperparameters. Zero weight modification guarantee.

    Usage:
        config = WatchConfig(output=OutputMode.CLI)
        watch = MoEWatch(model, config)
        watch.attach(trainer)
        trainer.train()
        watch.stop()

    Or as context manager:
        with MoEWatch(model, config) as watch:
            watch.attach(trainer)
            trainer.train()
    """

  ATTRIBUTES
    model: torch.nn.Module
      Reference to the MoE model being monitored (immutable)

    config: WatchConfig
      Configuration object (read-only after init)

    hook_manager: HookManager
      Manages all forward/backward hooks

    stat_collector: StatCollector
      Aggregates routing and gradient events

    baseline_tracker: BaselineTracker
      Tracks intervention-conditioned baselines

    entropy_analyzer: EntropyAnalyzer
      Tier 2 signal detection

    collapse_detector: CollapseDetector
      Expert health state tracking

    gradient_analyzer: GradientStarvationAnalyzer
      Tier 1 signal detection (v0.2.0)

    cross_layer_analyzer: CrossLayerCorrelation
      Tier 3 signal detection (v0.2.0)

    risk_fuser: RiskScoreFuser
      Fuses signals into risk_score (v0.2.0)

    intervention_engine: InterventionEngine
      Applies interventions to running training (v0.2.0)

    policy: PolicyBase (RulePolicy or BanditPolicy)
      Selects which intervention to apply (v0.2.0)

    alerts: list[Alert]
      History of all alerts emitted this session

    _running: bool
      Internal flag indicating monitoring state

    _global_step: int
      Current training step counter

  METHODS

    __init__(model: torch.nn.Module, config: WatchConfig = None)
      Args:
        model (torch.nn.Module) — MoE model to monitor
        config (WatchConfig) — configuration (uses defaults if None)
      Returns:
        None
      Raises:
        ValueError if model has no detectable MoE routers

    start() → None
      Args: None
      Returns: None
      Docstring: "Initialize hooks and collectors. Must be called before attach()."
      Side effects: Attaches hooks to detected router modules

    stop() → None
      Args: None
      Returns: None
      Docstring: "Detach all hooks and cleanup resources. Call at end of training."
      Side effects: Removes all registered hooks, stops monitoring

    attach(trainer: transformers.Trainer) → "MoEWatchCallback"
      Args:
        trainer (transformers.Trainer) — HuggingFace Trainer instance
      Returns:
        MoEWatchCallback — callback object for internal use
      Docstring: "Register MoEWatch as a HuggingFace training callback."
      Side effects: Calls trainer.add_callback()

    step(global_step: int) → WatchReport
      Args:
        global_step (int) — current training step number
      Returns:
        WatchReport — report with current risk scores, alerts, interventions
      Docstring: "Process one training step. Called by callback at each step."
      Side effects:
        - Collects latest stats from StatCollector
        - Runs all analyzers
        - Fuses signals into risk_score
        - Selects and applies interventions if enabled
        - Emits alerts if thresholds crossed

    get_alerts(since_step: int = 0) → list[Alert]
      Args:
        since_step (int) — return only alerts after this step (default 0)
      Returns:
        list[Alert] — filtered alert history
      Docstring: "Retrieve alert history since given step."

    __enter__() → "MoEWatch"
      Returns: self
      Docstring: "Context manager entry. Calls start()."
      Side effects: Calls start()

    __exit__(exc_type, exc_val, exc_tb) → None
      Returns: None
      Docstring: "Context manager exit. Calls stop() regardless of exception."
      Side effects: Calls stop()

CLASS: MoEWatchCallback
────────────────────────────────────────────────────────────────────────────────
  DOCSTRING: "HuggingFace Trainer callback that calls MoEWatch.step() after each step."

FILE: moewatch/_audit.py

PURPOSE
  Offline post-training diagnostic function. Accepts a trained model and
  validation dataloader, runs N forward passes, collects routing statistics,
  runs the full analyzer suite, and returns a comprehensive AuditReport.
  No hooks left attached after the function returns.

FUNCTION: audit
────────────────────────────────────────────────────────────────────────────────

  SIGNATURE
    def audit(
        model: torch.nn.Module,
        dataloader: torch.utils.data.DataLoader,
        num_batches: int = 100,
        config: WatchConfig | None = None,
        device: str = "cpu"
    ) → AuditReport

  DOCSTRING
    """
    Run offline diagnostic audit on a trained MoE model.

    Collects routing statistics over num_batches forward passes,
    runs entropy analysis, collapse detection, gradient estimation,
    and cross-layer analysis. Returns comprehensive report.

    Args:
        model: MoE model to audit
        dataloader: validation dataloader
        num_batches: how many batches to process (default 100)
        config: WatchConfig (uses defaults if None)
        device: "cpu" or "cuda"

    Returns:
        AuditReport with all analysis results

    Raises:
        ValueError if model has no detectable routers
        RuntimeError if device unavailable
    """

  PARAMETERS
    model: torch.nn.Module
      Input MoE model to audit

    dataloader: torch.utils.data.DataLoader
      Validation set dataloader

    num_batches: int = 100
      Number of batches to process (CPU-efficient limit)

    config: WatchConfig | None = None
      Configuration (uses WatchConfig defaults if None)

    device: str = "cpu"
      Torch device string

  RETURNS
    AuditReport — structured result containing:
      - per-layer entropy analysis
      - expert health state machine results
      - gradient starvation assessment
      - cross-layer propagation analysis
      - risk scores for each layer
      - summary statistics

  SIDE EFFECTS
    - Temporarily attaches hooks to model
    - Runs forward passes (no backprop)
    - Detaches all hooks at end (guaranteed via finally block)
    - Model weights unchanged

  IMPLEMENTATION NOTES
    - Use HookManager for temporary hook attachment/detachment
    - Use StatCollector to aggregate stats
    - Disable gradients: with torch.no_grad(): ...
    - Move data to device during loop
    - Wrap all analysis in try/finally to guarantee cleanup

DEPENDENCIES
  → moewatch.config (WatchConfig)
  → moewatch.hooks.manager (HookManager)
  → moewatch.collector.stat_collector (StatCollector)
  → moewatch.analyzer (all analyzers)
  → moewatch.report.audit_report (AuditReport)
  → torch

FILE: moewatch/hooks/manager.py
────────────────────────────────────────────────────────────────────────────────

PURPOSE
  HookManager owns all hook handles. Manages lifecycle: attach() registers
  hooks, detach() cleanly removes them. Guaranteed no hook leaks even on
  exception (uses finally blocks).

CLASS: HookManager
────────────────────────────────────────────────────────────────────────────────

  ATTRIBUTES
    model: torch.nn.Module
      Reference to monitored model

    stat_collector: StatCollector
      Destination for hook event writes

    config: WatchConfig
      Configuration reference

    _handles: list[torch.utils.hooks.RemovableHandle]
      List of all registered hook handles (for cleanup)

    _attached: bool
      Flag indicating attachment state

  METHODS

    __init__(model: torch.nn.Module, stat_collector: StatCollector, config: WatchConfig)
      Args:
        model (torch.nn.Module) — MoE model
        stat_collector (StatCollector) — event destination
        config (WatchConfig) — configuration
      Returns: None

    attach() → None
      Args: None
      Returns: None
      Docstring: "Attach all hooks to detected router modules."
      Side effects:
        - Auto-detects MoE routers via detect_router_modules()
        - Registers RouterForwardHook on each router
        - Registers GradientStarvationHook on expert weights (if enabled)
        - Stores all handles in self._handles
      Raises: ValueError if no routers detected

    detach() → None
      Args: None
      Returns: None
      Docstring: "Remove all registered hooks. Safe even if called multiple times."
      Side effects:
        - Iterates through self._handles and calls .remove()
        - Clears self._handles list
        - Sets self._attached = False
      Note: Uses try/finally to guarantee cleanup

    is_attached() → bool
      Args: None
      Returns: bool — True if hooks currently attached
      Docstring: "Check current attachment state."

DEPENDENCIES
  → moewatch.hooks.router_hook (RouterForwardHook)
  → moewatch.hooks.gradient_hook (GradientStarvationHook) [v0.2.0]
  → moewatch.hooks.detection (detect_router_modules)
  → moewatch.collector.stat_collector (StatCollector)
  → moewatch.config (WatchConfig)
  → torch

IMPLEMENTATION NOTES
  - Store handles so we can remove them later
  - Use try/finally in attach() to guarantee detach() on exception
  - Make detach() idempotent (safe to call multiple times)

PURPOSE
  Forward hook on router modules. Captures routing output tensor and constructs
  RoutingEvent with metadata. Writes to StatCollector. Designed for hot path:
  minimal allocation, no loops over tokens.

DATACLASS: RoutingEvent
────────────────────────────────────────────────────────────────────────────────
  Fields:
    timestamp: float
      Unix timestamp of event

    global_step: int
      Training step number

    layer_name: str
      Router module name (e.g., "layers.5.moe")

    routing_logits: torch.Tensor
      Raw router logits (shape: [batch_size, n_experts] or similar)

    selected_experts: torch.Tensor
      Top-k selected expert indices (shape: [batch_size, k])

    expert_count: int
      Number of experts in this router

    batch_size: int
      Current batch size

CLASS: RouterForwardHook
────────────────────────────────────────────────────────────────────────────────

  DOCSTRING
    """
    Forward hook on MoE router module. Captures routing decisions and
    metadata at each forward pass. Zero-copy when possible.
    """

  METHODS

    __init__(layer_name: str, stat_collector: StatCollector, config: WatchConfig)
      Args:
        layer_name (str) — name of router module (e.g., "layers.5.moe")
        stat_collector (StatCollector) — writes events here
        config (WatchConfig) — configuration reference
      Returns: None

    __call__(module: torch.nn.Module, input: tuple, output: torch.Tensor) → None
      Args:
        module (torch.nn.Module) — router module being hooked
        input (tuple) — module input
        output (torch.Tensor) — router output (logits or indices)
      Returns: None
      Docstring: "Hook callback. Captures routing output and writes RoutingEvent."
      Side effects:
        - Infers expert count from output shape
        - Constructs RoutingEvent
        - Calls stat_collector.write_routing_event()

PURPOSE
  Cumulative Sum (CUSUM) change-point detection. Used by entropy and gradient
  analyzers to detect drift in signal sequences. Replaces naive recent-vs-prior
  thresholding with statistically principled approach.

FUNCTION: detect_change
────────────────────────────────────────────────────────────────────────────────

  SIGNATURE
    def detect_change(
        series: list[float] | numpy.ndarray,
        threshold: float = 5.0,
        drift: float = 1.0
    ) → tuple[bool, int, float]

  DOCSTRING
    """
    Detect change-point in time series using CUSUM algorithm.

    Args:
        series: sequence of values
        threshold: alert threshold for cumulative sum
        drift: minimum change magnitude to detect (default 1.0)

    Returns:
        (change_detected: bool, change_point_step: int, cusum_value: float)

    Algorithm:
        1. Compute CUSUM = cumulative sum of (x - drift)
        2. Track max CUSUM
        3. If CUSUM exceeds threshold: change detected
        4. Reset on detection
    """

  PARAMETERS
    series: list[float] | numpy.ndarray
      Time series values

    threshold: float = 5.0
      CUSUM threshold for alert

    drift: float = 1.0
      Expected drift magnitude (window size parameter)

  RETURNS
    tuple[bool, int, float]
      - change_detected: True if CUSUM crossed threshold
      - change_point_step: step where change started (or -1 if not detected)
      - cusum_value: current cumulative sum value

  ALGORITHM
    cusum_pos = 0
    for i, val in enumerate(series):
        cusum_pos = max(0, cusum_pos + (val - drift))
        if cusum_pos > threshold:
            return (True, i, cusum_pos)
    return (False, -1, cusum_pos)

────────────────────────────────────────────────────────────────────────────────

  ATTRIBUTES
    threshold: float
      Alert threshold

    drift: float
      Expected drift magnitude

    _cusum_pos: float
      Current positive cumulative sum

    _min_cusum: float
      Minimum cusum value (for negative detection)

  METHODS

    __init__(threshold: float = 5.0, drift: float = 1.0)
      Args:
        threshold (float) — alert threshold
        drift (float) — drift parameter
      Returns: None

    update(value: float) → bool
      Args:
        value (float) — new series value
      Returns: bool — True if change detected
      Docstring: "Process one value, return True if threshold crossed."

    reset() → None
      Args: None
      Returns: None
      Docstring: "Reset internal state."

DEPENDENCIES
  → numpy (for array operations)

IMPLEMENTATION NOTES
  - CUSUM formula: S+ = max(0, S+ + (x - drift))
  - When S+ > threshold: change detected, reset to 0
  - Drift parameter tunes sensitivity (lower drift = more sensitive)
  - Common choices: drift=1.0 for normalized signals, drift=0.5 for sensitive detection

PURPOSE

CLASS: BanditPolicy (extends PolicyBase)
────────────────────────────────────────────────────────────────────────────────

  DOCSTRING
    """
    Phase 2 contextual bandit policy. Learns which interventions work
    at which training stages.

    State: (risk_score, layer_id, step, intervention_history)
    Action: {NoOp, AuxLoss, RouterNoise, ExpertDropout}
    Reward: discounted counterfactual ΔH over observation window
    Algorithm: epsilon-greedy exploration with Q-learning value updates
    """

  ATTRIBUTES
    config: WatchConfig
      Configuration (epsilon, gamma, etc.)

    memory: PolicyMemory
      Experience replay buffer

    _q_values: dict[str, dict[str, float]]

      Args:

      Returns: None

    select_action(state: PolicyState) → InterventionAction
      Args:
        state (PolicyState) — current state
      Returns:
        InterventionAction — selected action
      Docstring:

        explore (random action). Else exploit (best Q-value)."
      Algorithm:
        1. Hash state to state_key
        2. Get Q-values for all actions in this state
        3. If random() < epsilon: select random action

      Returns: None
      Docstring: "Update Q-values and policy memory with new experience."
      Side effects:
        - Stores (state, action, reward) in memory
        - Updates Q(state, action) ← Q + α * (reward - Q)
        - Increments action count for this state-action pair

    save_checkpoint(path: str) → None
      Args:
        path (str) — file path to save to
      Returns: None
      Docstring: "Save Q-values and memory to JSON."

      Args:

      Returns: None

  - Epsilon decay: optionally decrease epsilon over time
  - State hashing: convert PolicyState to string key for Q-table lookup

FILE: moewatch/policy/reward.py
────────────────────────────────────────────────────────────────────────────────

PURPOSE
  Computes discounted counterfactual reward for completed intervention.
  reward = Σ ΔH_counterfactual(t+k) * γ^k for k=1..K. ΔH_counterfactual
  from BaselineTracker. Exponential discount factor γ (default 0.95).
  Observation window K (default 50 steps).

CLASS: RewardComputer
────────────────────────────────────────────────────────────────────────────────

  DOCSTRING
    """
    Computes discounted counterfactual reward for interventions.

    reward = Σ ΔH_counterfactual(t+k) * γ^k   for k = 1 to K

    where:
      ΔH_counterfactual = entropy_actual - entropy_baseline
      γ = discount factor (typically 0.95)
      K = observation window (typically 50 steps)

    Positive reward = intervention helped routing stability.
    """

  ATTRIBUTES
    config: WatchConfig
      Configuration (gamma, window_steps)

    baseline_tracker: BaselineTracker
      Source of counterfactual baseline values

  METHODS

    __init__(config: WatchConfig, baseline_tracker: BaselineTracker)
      Args:
        config (WatchConfig) — configuration
        baseline_tracker (BaselineTracker) — baseline source
      Returns: None

    compute_reward(
        layer_name: str,
        entropy_values: list[float],
        start_step: int,
        end_step: int
    ) → float
      Args:
        layer_name (str) — target layer
        entropy_values (list[float]) — entropy values over observation window
        start_step (int) — step when intervention applied
        end_step (int) — step when observation window closed
      Returns:
        float — discounted counterfactual reward
      Docstring:
        "Compute reward as discounted sum of counterfactual deltas."
      Algorithm:
        1. For k = 1 to K:
           - delta_k = entropy_values[k] - baseline_tracker.get_baseline(layer, start+k)
           - reward += delta_k * (gamma ** k)
        2. Return reward

DEPENDENCIES
  → moewatch.collector.baseline_tracker (BaselineTracker)
  → moewatch.config (WatchConfig)
  → numpy

IMPLEMENTATION NOTES
  - Discount factor γ typically 0.95 (later steps contribute less)
  - Window K typically 50 steps (observation period)
  - ΔH_counterfactual computed as actual - baseline (positive = improvement)
  - Handle missing baseline: use 0 delta if baseline not available

KEY DESIGN PRINCIPLES & ARCHITECTURE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. SIGNAL TIER HIERARCHY (Empirically Ordered)
   ────────────────────────────────────────────
   Tier 1: Gradient Starvation (earliest, 50–200 steps before collapse)
   Tier 2: Entropy Drift (intermediate, 30–100 steps before)
   Tier 3: Cross-Layer Correlation (localization, system-level)

   Risk Score: Weighted fusion via RiskScoreFuser
               risk = 0.6*T1 + 0.3*T2 + 0.1*T3

2. INTERVENTION-CONDITIONED COUNTERFACTUAL BASELINE
   ─────────────────────────────────────────────────
   Critical for causal attribution. BaselineTracker maintains rolling
   linear regression on "clean" history (pre-intervention steps only).
   Prevents baseline contamination by excluding post-intervention windows.

   Reward = actual_signal - baseline_projected
   Used by policy for learning which actions work.

3. TWO-PHASE POLICY ARCHITECTURE
   ──────────────────────────────
   Phase 1 (RulePolicy): Deterministic, threshold-based. Stable baseline.
   Phase 2 (BanditPolicy): Learning contextual bandit. Improves within run.

   Both share PolicyBase interface. Config switches via policy_type.

4. SAFETY-FIRST INTERVENTION
   ──────────────────────────
   SafetyGuard enforces pre-flight checks:
     - Cooldown: spacing between interventions
     - Delta limit: bounded magnitude changes
     - Loss guard: freeze on training loss spike
     - Neighbor check: adjacent layer stability

   Failed checks → downgrade to NoOp (don't abort).

5. ZERO WEIGHT MODIFICATION GUARANTEE
   ────────────────────────────────────
   Hooks are read-only. Modifications go through trainer API only.
   This protects against accidental model corruption.

6. THREAD-SAFE EVENT COLLECTION
   ──────────────────────────────
   RingBuffer + StatCollector + Lock = safe concurrent access.
   Necessary for real-time monitoring during training.
